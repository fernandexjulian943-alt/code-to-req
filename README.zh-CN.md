# Code-to-Req: AI 代码架构审计引擎

> 将任意代码仓库转化为专业的架构审计报告 —— 包含架构图、知识图谱和结构化需求文档。

**为什么不直接问 AI？** 这个 Skill 产出的是标准化、多文档的审计报告，不是聊天回复。包含架构图（Mermaid）、知识图谱（JSON + 可视化）、健康度评分和可执行的需求建议。输出可直接用于汇报，不只是开发笔记。

## 输出示例

```
📦 docs/code-analysis/
├── README.md                  ← 报告入口（健康度评分 + 关键发现）
├── project-profile.md         ← 项目画像（技术栈/规模/模块清单）
├── architecture-diagrams.md   ← 5 张架构图（系统层级/模块依赖/数据流/时序/ER）
├── knowledge-graph.md         ← 知识图谱（实体/关系/JSON 数据）
├── reading-plan.md            ← 智能阅读计划
├── requirements-draft.md      ← 需求草稿
└── tech-proposal.md           ← 技术方案
```

## 安装

**方式 A：Plugin marketplace**
```
/plugin install code-to-req
```

**方式 B：手动安装**
```bash
git clone https://github.com/YOUR_ORG/code-to-req.git ~/.claude/skills/code-to-req
```

## 使用

```bash
# 完整流程：扫描 → 图表 → 分析 → 多角色讨论 → 输出报告
/code-to-req <需求描述>

# 快速扫描：项目画像 + 架构图 + 知识图谱
/code-to-req scan

# 扫描 + 规划：生成针对特定需求的阅读计划
/code-to-req plan <需求描述>
```

## 推荐模型

| 场景 | 推荐模型 | 说明 |
|------|---------|------|
| **最佳体验** | Claude Opus 4.6+ | 多角色讨论深度好，架构分析准确 |
| **性价比** | Claude Sonnet 4.6 | 全流程可用，讨论深度略降 |
| **国内首选** | DeepSeek V4 Pro | 代码理解强，中文输出好，成本低 |
| **国内备选** | Qwen 3.6 Plus | 中文理解最强，代码能力稍弱 |
| **不建议** | Haiku / 小参数模型 | 多角色讨论质量不足 |

无需额外配置 API Key，Skill 使用你当前 Claude Code 会话的模型。

## 特性

- **智能缩放**：根据项目规模（<100 到 1000+ 文件）自适应策略
- **5 种架构图**：系统层级、模块依赖、数据流、API 时序、ER 图
- **知识图谱**：Mermaid 可视化 + JSON 数据（可接入 sigma.js/D3 渲染）
- **健康度评分**：5 个维度 A/B/C/D 量化评级
- **多角色分析**：产品经理 + 架构师 + 开发工程师三视角交叉讨论
- **大项目支持**：智能过滤、采样扫描、分批读取

## 工作流程

```
阶段 0：规模评估 → 选择扫描策略
阶段 1：代码扫描（无 AI，纯静态分析）
阶段 1.5：架构图 + 知识图谱生成
阶段 2：智能阅读规划（AI 驱动）
阶段 3：渐进式代码读取 + 理解
阶段 4：多角色协作讨论（PM / 架构师 / 开发）
阶段 5：报告生成
```

## 适合谁

- **产品经理**：接手老系统，快速理解全貌，产出改进需求
- **技术负责人**：新项目评估，快速出架构报告
- **架构师**：系统改造前的现状审计
- **新人入职**：比看 wiki 快 10 倍的项目理解方式
